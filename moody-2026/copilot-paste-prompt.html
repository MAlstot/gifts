    PATH C · KEEP IT
    
Save it as a reusable Copilot skill

    
Ran it once and want it on tap? Copilot can save a workflow like this as a skill — a named recipe in four parts: Skill Name · Input · Workflow · Constraints. Save it wherever you build agents (Copilot Studio / agent instructions), or keep it pinned in a doc to paste on demand.


    

      See the four-part recipe in a simple worked example
      
Skill Name: Assignment Alignment Reviewer
Input:
- Assignment description
- Learning outcomes
Workflow:
1. Extract assignment requirements.
2. Compare requirements to outcomes.
3. Identify gaps.
4. Generate feedback questions.
5. Output an alignment report.
Constraints:
- Do not rewrite faculty materials.
- Provide feedback only.

      
Why it works: the Input names exactly what to paste, the Workflow is numbered thinking steps, and the Constraints draw the boundary — feedback, never rewriting. Constraints are to your skill what Rails are to student prompts: write them first.

      Blank recipe: build a skill of your own
      
Skill Name: [Name it — e.g., Case Study Feedback Coach]
Input:
- [What you'll paste in — e.g., the assignment as students see it]
- [Context it needs — e.g., learning outcomes, rubric, your continuum level]
Workflow:
1. [Read/extract — what should it pull out first?]
2. [Compare/analyze — the thinking step]
3. [Identify — gaps, risks, or misalignments]
4. [Generate — questions or feedback, not rewrites]
5. [Output — name the report format you want back]
Constraints:
- Do not rewrite faculty materials; provide feedback only.
- [Your boundary — matched to your assignment's continuum level]
- Never include identifiable student data.
